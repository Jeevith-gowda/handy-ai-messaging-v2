## đź—„ď¸Ź Database Schema (Mongoose Models)

### `Customer`
Tracks clients and leads.
- `name` (String, default: 'Unknown')
- `firstName` / `lastName` (String)
- `phone` (String, unique)
- `email` (String)
- `isSpam` (Boolean) - Controlled by AI triage.

### `Message`
The single source of truth for all omni-channel interactions.
- `conversationId` / `customerId` (Optional Ref) - *Null if spam.*
- `from` / `fromPhone` / `to` (String)
- `body` (String)
- `direction` (Enum: `inbound`, `outbound`)
- `mediaUrls` ([String])
- `isRead` (Boolean, default: false)
- `isCall` (Boolean)
- `callId` / `callStatus` / `callDuration` / `callSummary` (Fields for call tracking)
- `aiDraft` (String) - Autogenerated reply.
- `isSpam` (Boolean)

### `User` (Staff / Admin)
Handles internal authentication.
- `name`, `email` (Unique), `password` (auto-bcyrpt hashed via pre-save hook)
- `role` (Enum: `admin`, `handyman`)
- *Handyman Specifics*: `skills`, `hourlyRate`, `availability`

### `Project`
Tracks active construction tasks.
- Links `Customer` to `Handyman`
- `status` (Enum: `pending`, `accepted`, `in_progress`, etc.)

## đź¤– The Multi-Agent Architecture
The platform is designed around a dedicated fleet of AI agents working dynamically behind the scenes.

1. **Agent Alpha (The Supervisor)**: 
   - Oversees subordinates and updates the "Control Room" UI.
   - Evaluates project statuses, re-assigns quotes, and dictates overarching workflows.
2. **Agent 1 (The Intake Worker)**: 
   - Lives inside the webhook ingestion pipeline (`lib/messageHandler.js`).
   - Receives inbound webhooks, fetches the last 5 conversation messages for memory.
   - Detects and flags spam (so it never hits the CRM).
   - Drafts hyper-contextual replies for genuine users (Human-in-the-Loop review).

## âš™ď¸Ź Core Workflows

1. **OpenPhone Webhook Pipeline**: (Currently tunneled via `ngrok` for local dev)
   - Payload hits `POST /api/webhooks/openphone`.
   - Distinguishes between `message.received`, `call.completed`, and `call.summary.completed`.
   - Extracts numbers, downloads incoming MMS/media to Cloudinary.
2. **CRM Triage Logic**: 
   - Agent 1 intercepts the raw payload *before* database injection.
   - If spam: message saves as orphaned (`isSpam: true`, `customerId: null`).
   - If genuine: normalizes phone number, queries `Customer`, creates "New Prospect" if unknown, routes to Thread.
